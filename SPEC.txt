------------------------------------------------------------------------------
LOCKED DECISIONS
------------------------------------------------------------------------------
  1. Six branch bins (names and semantics are FIXED):
       scalar       ndim 1                       -> straight through
       vec1         ndim 2, length 1 everywhere  -> collapsed via [:, 0]
       jagged       ndim 2, variable length      -> exploded + padded
       jagged_deep  ndim > 2                     -> parquet only, never CSV
       empty        max_len 0 everywhere         -> skipped
       unreadable   uproot cannot decode         -> recorded by NAME + reason
                                                    only; that is good enough
  2. Canonical parquet = bulk storage. Lists stay lists, jagged_deep intact.
     CSVs are disposable views regenerable without re-touching ROOT.
  3. Two layers, two scripts:
       Layer 1  scan.py     census every struct -> parquet + manifest.json
                            + manifest.txt
       Layer 2  convert.py  ROOT -> flat CSV
                  2a QUICK      no pre-scan; scan+flatten one shot, default
                                policies == events.root behavior (hard
                                structs removed from CSV)
                  2b FROM-SCAN  manifest-driven; honors per-branch policy
                                edits in manifest.json; reads canonical
                                parquet, never re-reads ROOT
  4. Jagged policies (per branch, "policy" field in manifest.json):
       pad_max (default) | first:N | drop
  5. BASE-POLICY pad flag: -999 (numeric; jagged-derived columns stay
     numeric dtype). Chosen over -99/-255 as the entrenched HEP sentinel,
     far from physical ranges. --fill x restores the legacy events.root
     'x' sentinel (byte-parity with the original CSVs); --fill nan pads
     NaN; --fill <number> pads that number. NOTE: default output now
     DIVERGES from the April events.root CSVs unless --fill x is passed.
  6. Tree resolution: --tree, else auto-detect = the TTree/RNTuple with the
     most entries (fastframe files carry reco/truth/etc, not 'analysis').
     Both TTree and RNTuple read paths supported.
  7. --tag KEY=VALUE integer columns stamped on every row (kept from
     flatten_events).
  8. Headless-first: pip-only deps (uproot awkward pandas numpy pyarrow),
     argparse CLI, custom \r progress bars on every pass, zero display
     requirements. Deploy = git clone + pip install -r requirements.txt.
  9. Any served pages bind 127.0.0.1 ONLY; ssh tunnel is the only door.
     Port/tunnel hygiene checklist lives in README ("before you log out").

------------------------------------------------------------------------------
KNOWN LIMITS / ROADMAP  (deliberately deferred)
------------------------------------------------------------------------------
  - Pass 1 caches all branch arrays in RAM (transplanted behavior). Fine
    for sample-scale files; big files want bigmem3 or a future chunked mode.
  - CSV assembled as one in-RAM DataFrame; chunked CSV writes are roadmap.
  - pad_p99 / overflow-flag policy: roadmap (first:N covers most of it).
  - Outlier fan-out warning: manifest exposes len_med/len_p95/len_max so
    column explosions are visible BEFORE committing to a flatten.
