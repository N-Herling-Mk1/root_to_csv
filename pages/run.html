RUN ORDER — two paths

Both paths end at the same flat CSV. Path A is one command with sane defaults. Path B puts a curation step in the middle so you control exactly how each jagged branch unfolds.

PATH A — QUICK (2a · no pre-scan)
  1. Run convert.py on the .root file.
  2. Done. Scan + flatten happen in one shot with default policies — the original events.root behavior.
python3 convert.py /data2/kjohns/run3_sample_fastframe_files/<file>.root
PATH B — CURATED (2b · scan, edit, build)
  1. scan.py the file → census + parquet + manifests.
  2. Read <name>_manifest.txt; edit per-branch "policy" in <name>_manifest.json.
  3. convert.py --from-scan → CSV built from the parquet. ROOT is never re-read.
python3 scan.py <file>.root --name s1 --out ./s1_scan # edit ./s1_scan/s1_manifest.json ("policy" per jagged branch) python3 convert.py --from-scan ./s1_scan
