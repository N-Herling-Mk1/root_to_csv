TIER 1 — command line · headless server

Everything over ssh (e.g. @atlng02.physics.arizona.edu, bigmem3). Pip-installable python — no ATLAS environment, no ROOT build, no display, no open ports.

Prerequisites: python 3.9+ and pip (python3 --version).

THE COMMANDS — INSTALL · PRE-SCAN · DECIDE · CONVERT

# 1 · install (once) git clone https://github.com/N-Herling-Mk1/root_to_csv.git cd root_to_csv pip install --user -r requirements.txt # 2 · pre-scan → writes ./s1_scan/ (parquet + manifest.json + manifest.txt) python3 scan.py <file>.root --name s1 --out ./s1_scan # 3 · decide → set "policy" per jagged branch: pad_max | first:N | drop nano ./s1_scan/s1_manifest.json # 4 · convert → writes s1_flat.csv + s1_build.txt into ./s1_scan/ python3 convert.py --from-scan ./s1_scan # QUICK PATH — skip the pre-scan: install → CSV in one command (scan runs internally) python3 convert.py <file>.root

SAMPLE OUTPUT — THE SYNOPSIS LINES

NOTES — WHICH NAMES ARE YOURS
YOURS <file>.root — your input ROOT file, wherever it lives.
YOURS s1 — the sample name, set by --name. Every output inherits it: s1_scan/, s1_flat.csv, s1_manifest.json… Skip --name and the ROOT file's stem is used.
YOURS ./s1_scan — derived from the name automatically; pass --out DIR only to put it somewhere else.
FIXED the suffixes — _flat.csv · _canonical.parquet · _manifest.json/.txt · _build.txt — the toolkit's, always.
FIXED scan.py · convert.py — the programs themselves.

WHAT PRODUCES WHAT

THE BASE POLICY

Built in — it's what you get by never touching manifest.json (or taking the quick path). Every branch follows its default:

  • jagged → pad_max — explode to the longest event, pad with -999
  • scalar → as-is  ·  vec1 → collapsed
  • hard structs → per the fate table below

Run the steps in sequence and take what comes out — step 3 is simply a no-op. Editing policies is opt-in, never required.

WHAT CONVERT.PY DOES WITH EACH BIN — [FATE TABLE]

BININ THE CSV?FATE
scalar ✓one column, as-is
vec1 ✓collapsed to one column via [:,0]
jagged ✓exploded to _0…_N, padded with -999 — your policy sets N
jagged-deep ✗skipped — kept intact in the parquet
empty ✗skipped — listed in the manifest
unreadable ✗skipped — name + typename + reason recorded

Quick mode applies exactly this table with default policies. Nothing is dropped silently — what skips the CSV is in the parquet or named in the manifest.

READING THE PRE-SCAN — WHAT TO DO WITH THE NUMBERS

SIGNAL IN MANIFEST.TXTIT MEANSDO THIS IN MANIFEST.JSON
max ≫ med on a jagged line one fat outlier event is setting the column count "policy": "first:N" with N near p95
low fill fraction column block would be mostly sentinels "policy": "drop"
branch listed under jagged-deep / unreadable can't be CSV columns at all nothing — already parked in parquet / recorded by name

WHERE EVERYTHING LIVES & HOW TO USE IT

One directory per sample — default ./<name>_scan/, created where you run the command (never next to the source ROOT file; --out DIR overrides).

FILEUSE IT WITHWHAT IT'S FOR
<name>_flat.csv pandas.read_csv The deliverable — feeds analysis / NN training.
<name>_canonical.parquet awkward.from_parquet Bulk storage, lists intact — regenerate CSVs anytime without ROOT.
<name>_manifest.txt less / Ctrl-F Human audit: six-bin summary + per-branch stats.
<name>_manifest.json any editor The control surface — "policy" drives step 4.
<name>_build.txt any editor Receipt: CSV shape/size, fill, policy-dropped branches.
Tier 1 opens no ports and no tunnels — there is nothing to clean up when you log out.
THE OUTPUT FILES — WHAT'S INSIDE
s1_flat.csv
  • one row per event
  • your --tag KEY=VALUE int columns first
  • scalars, then vec1 (collapsed via [:,0])
  • jagged as name_0 … name_N — policy sets N
  • -999 wherever an event ran short
s1_canonical.parquet
  • every readable branch, lists kept as lists
  • jagged-deep stored intact
  • the source --from-scan rebuilds from — ROOT never re-read
s1_manifest.json
  • one entry per readable branch:
  •   bin (category), C++ + awkward types
  •   len min·med·p95·max, fill fraction
  •   sample values, projected fan-out
  •   "policy" — the slot you edit
  • unreadables in their own list: name + typename + reason
s1_manifest.txt
  • six-bin tallies for the whole file
  • per-branch stat lines (grep-able)
  • projected sentinel insertions
  • condensed roster of EVERY branch — flattened vs unreadable
s1_build.txt
  • rows × cols of the CSV, file size
  • fill flag used
  • policy-dropped branches, by name
  • build timing
