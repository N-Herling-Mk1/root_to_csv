OUTPUTS — what lands on disk

A scan (or quick convert) drops everything into one output directory, default ./<name>_scan/

FILEWHAT IT IS
<name>_flat.csv The flat CSV. Jagged branches exploded to _0…_N columns, pads filled with the sentinel. This is the file that feeds pandas / NN training.
<name>_canonical.parquet Bulk storage. Everything readable, lists kept as lists, jagged-deep intact. CSVs are disposable views — regenerate them from this without ever re-reading ROOT.
<name>_manifest.txt Human-readable audit: the six-bin summary, per-branch census with length stats, jagged-deep structure detail, and a condensed flattened-vs-unreadable list. Ctrl-F any branch name.
<name>_manifest.json Machine-readable census. The "policy" field per jagged branch (pad_max · first:N · drop) is yours to edit; --from-scan obeys it.
<name>_build.txt Written by --from-scan builds: CSV shape/size, fill used, and which branches the policies dropped.
