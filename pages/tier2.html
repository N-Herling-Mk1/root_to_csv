TIER 2 — VS Code via tunneling

code-server on the headless node, reached through an ssh tunnel in your browser. Everything binds 127.0.0.1 — the tunnel is the only door.

Tier 2 in the wild: code-server on atlng02 at localhost:8080, repo tree in the Explorer, integrated terminal below. Every Tier 1 command runs unchanged in that terminal; outputs download with a right-click.

2 · CODE-SERVER (VS CODE IN THE BROWSER)

Check for it: command -v code-server && code-server --version. Install without root if missing:

curl -fsSL https://code-server.dev/install.sh | sh -s -- --method=standalone --prefix=$HOME/.local export PATH="$HOME/.local/bin:$PATH"

Run it (loopback only), then tunnel from your local machine:

# on the server code-server --bind-addr 127.0.0.1:8080 # on your local machine, new terminal ssh -L 8080:localhost:8080 user@server

Browse to http://localhost:8080. Password: auto-generated at first run, stored on the server in ~/.config/code-server/config.yaml:

cat ~/.config/code-server/config.yaml

Download any file: right-click it in the Explorer sidebar → Download. Paste only bare commands into terminals — never lines that include a [user@host]$ prompt.

VIEW HTML OUTPUTS (tiny web server + tunnel)

# on the server — 127.0.0.1 bind ONLY python3 -m http.server 8888 --bind 127.0.0.1 # on your local machine ssh -L 8888:localhost:8888 user@server # browse: http://localhost:8888

WARNING — [FYI] · CLEAN UP TUNNEL/PROCESSES

Before you log out: rule zero is loopback-only binds, so the tunnel is the only door — then close up anyway.

Never bind 0.0.0.0 on a shared server. If a command or tutorial suggests it, use 127.0.0.1 instead and tunnel.
  • Audit listeners — empty output = clean:
    lsof -iTCP -sTCP:LISTEN -a -u $USER
  • Kill anything left over (e.g. a forgotten http.server or code-server):
    kill <PID>
  • Optional — clear lingering VS Code remote processes:
    pkill -u $USER -f vscode-server
  • Local side — a foreground ssh -L tunnel dies with its terminal. Find stray backgrounded ones:
    ps aux | grep '[s]sh -L'
