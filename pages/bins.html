THE SIX BINS

Pass 1 reads every branch and files it into exactly one of six bins. The bin decides its fate in the flat CSV. Colors run cool → hot with structural difficulty.

SCALAR detection: ndim == 1 CSV: straight through

One value per event. The easy case — becomes a single CSV column unchanged. Counter branches auto-written by ROOT (njet_pt and friends) land here too. Struct-typed scalars that would crash pandas are detected and stringified safely.

eventNumber → 4021 met → 37.2
VEC1 detection: ndim == 2, length == 1 in every event CSV: collapsed to scalar

A vector that always holds exactly one element — common after a one-vertex selection, where per-vertex quantities are length-1 lists. Collapsed via [:, 0] into a plain column, no suffix, no padding. If even one event has length 0 or 2+, the branch is jagged instead.

msvtx_eta → [1.24] → 1.24
JAGGED detection: ndim == 2, variable length CSV: exploded + padded

A variable-length vector per event — the defining ATLAS shape. Exploded into name_0 … name_{max-1} columns; events shorter than the maximum are padded with the flag value -999 by default (numeric — columns stay numeric; --fill x restores the legacy sentinel, --fill nan pads NaN).

The catch: column count is set by the single fattest event in the file. One event with 200 tracks → 200 columns, even if the median is 5. The manifest's len_med / len_p95 / len_max shows this before you build; the first:N and drop policies tame it.

jet_pt → [45.2, 12.1] (max_len 4) → jet_pt_0=45.2 jet_pt_1=12.1 jet_pt_2=-999 jet_pt_3=-999
JAGGED-DEEP detection: ndim > 2 parquet only — never CSV

A list of lists per event — e.g. per-vertex, per-cluster times: variable vertices per event, variable clusters per vertex. Not a tensor (no fixed shape); jagged-of-jagged. Fully flattening one is combinatorial column explosion, so it is never written to CSV. It is preserved intact in the canonical parquet, and the manifest records its full structure: ndim, per-axis max lengths, leaf dtype, and the hypothetical column count you were spared.

msvtx_cluster_time → [[t00, t01], [t10]] ndim=3 axes: 2 × 2
EMPTY detection: max length == 0 in every event skipped

The branch exists but never holds a value in this sample — usually a selection upstream drained it. Nothing to write; skipped from the CSV and listed in the manifest so its absence is documented, not mysterious.

some_branch → [] [] [] … (all events)
UNREADABLE detection: uproot raises on read name + reason recorded — nothing else possible

Custom C++ classes uproot has no interpretation for. Historical offenders in events.root: InputMapNN1b, caloClusterEnergies, msegNCB_counts. The manifest records the branch name, C++ typename, and failure reason — recording the name is all that can be done, and it's good enough. Nothing is lost silently.

InputMapNN1b type: map<…> reason: no interpretation
