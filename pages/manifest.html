MANIFEST.JSON — the control surface

The pre-scan writes it; step 4 obeys it. One file, one editable field: "policy". Everything else is evidence.

THE SHAPE — WHAT YOU'RE LOOKING AT

// "meta" — identity of the scan (read-only) { "meta": { "name": "v1", "source": ".../HSS_mH125_mS55_ct5320_537840_mc23e_fullsim.root", "tree": "analysis", // auto-detected "n_events": 423, "tool_version": "0.1.0" }, "tags": {}, // one entry per readable branch — evidence, plus the one dial "branches": { "L1MuRoI_eta": { "ak_type": "423 * var * float32", "category": "jagged", // its bin — read-only "dtype": "float32", "cpp_typename": "std::vector<float>", "len_min": 0, "len_med": 2.0, "len_p95": 6.0, // the decision numbers "len_max": 41, "fill_frac": 0.62, "fanout": 41, // columns this makes as-is "samples": ["[0.83, -1.2, ...]"], "policy": "pad_max" // <<< THE FIELD YOU EDIT }, "HLTmuRoIClus_eta": { "category": "vec1", "policy": "collapse" // machine-set — leave it } }, // couldn't be read — empty on clean files, never silent "unreadable": [] }
The one rule: you only ever edit "policy" values, and only on jagged branches. Other bins carry machine-set policies (collapse on vec1, etc.) — leave those. The rebuild reads nothing else back; editing the stats changes nothing.

THE THREE EDITS — WORKED EXAMPLES

CAP AN OUTLIER · first:N

The stats say len_med 2 · len_p95 6 · len_max 41 — one fat event is minting 412 columns.

"policy": "pad_max" "policy": "first:6"

Columns go L1MuRoI_eta_0…_40 → L1MuRoI_eta_0…_5. Events longer than 6 are truncated; shorter ones pad -999.

DROP A GHOST · drop

"fill_frac": 0.03 — the block would be 97% pads.

"policy": "pad_max" "policy": "drop"

No columns in the CSV. The branch stays whole in the parquet and is named in build.txt — nothing silent.

STAND DOWN · pad_max

Changed your mind? Set it back — or never touch it.

"policy": "first:6" "policy": "pad_max"

The base policy: explode to the longest event, pad -999. Untouched manifest = quick-path output.

POLICY REFERENCE

POLICYCOLUMNS MADEREACH FOR IT WHEN
pad_max name_0 … name_(max-1) default — fan-out is sane, keep everything
first:N name_0 … name_(N-1) max ≫ med — outliers set the width; pick N near p95
drop none low fill — the block would be mostly -999

APPLY IT

python3 convert.py --from-scan ./s1_scan

Seconds, not minutes — the rebuild reads the parquet, never the ROOT file. Iterate on policies freely; every build's build.txt records shape, fill, and what was dropped, and the terminal synopsis shows the new column count.
