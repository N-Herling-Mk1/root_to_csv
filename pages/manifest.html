MANIFEST.JSON — the control surface

The pre-scan writes it; step 4 obeys it. One file, one editable field: "policy". Everything else is evidence.

THE SHAPE — WHAT YOU'RE LOOKING AT

// header — identity of the scan (read-only) { "name": "s1", "source": "PT_ovchan.root", "tree": "reco", "n_events": 3233, "tags": {"mH": 125, "mS": 35}, // one entry per readable branch — all evidence, except "policy" "branches": { "met": { "category": "scalar", // its bin — read-only "cpp_type": "float", "len": null, "fill": 1.0, "policy": null // scalars have no policy }, "jet_pt": { "category": "jagged", "cpp_type": "vector<float>", "len": {"min": 0, "med": 4, "p95": 9, "max": 412}, "fill": 0.87, "samples": [45.2, 12.1], "fanout": 412, // columns this makes as-is "policy": "pad_max" // <<< THE FIELD YOU EDIT } }, // couldn't be read — recorded, never silent (read-only) "unreadable": [ {"name": "InputMapNN1b", "cpp_type": "map<string,float>", "reason": "unsupported container"} ] }
The one rule: you only ever edit "policy" values, and only on jagged branches. The rebuild reads nothing else back — editing the stats changes nothing.

THE THREE EDITS — WORKED EXAMPLES

CAP AN OUTLIER · first:N

The stats say med 4 · p95 9 · max 412 — one fat event is minting 412 columns.

"policy": "pad_max" "policy": "first:9"

Columns go jet_pt_0…_411 → jet_pt_0…_8. Events longer than 9 are truncated; shorter ones pad -999.

DROP A GHOST · drop

"fill": 0.03 — the block would be 97% pads.

"policy": "pad_max" "policy": "drop"

No columns in the CSV. The branch stays whole in the parquet and is named in build.txt — nothing silent.

STAND DOWN · pad_max

Changed your mind? Set it back — or never touch it.

"policy": "first:9" "policy": "pad_max"

The base policy: explode to the longest event, pad -999. Untouched manifest = quick-path output.

POLICY REFERENCE

POLICYCOLUMNS MADEREACH FOR IT WHEN
pad_max name_0 … name_(max-1) default — fan-out is sane, keep everything
first:N name_0 … name_(N-1) max ≫ med — outliers set the width; pick N near p95
drop none low fill — the block would be mostly -999

APPLY IT

python3 convert.py --from-scan ./s1_scan

Seconds, not minutes — the rebuild reads the parquet, never the ROOT file. Iterate on policies freely; every build's build.txt records shape, fill, and what was dropped, and the terminal synopsis shows the new column count.
