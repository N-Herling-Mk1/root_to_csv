INSTRUCTIONS — fastest & full

Everything is pip-installable python. No ATLAS environment, no ROOT build, no display. Works the same on bigmem3, atlng01/02, a laptop, or WSL.

PREREQUISITES

python 3.9+ and pip. Check with python3 --version.

CLONE · INSTALL · SELF-TEST  (once, both options)

git clone https://github.com/N-Herling-Mk1/root2csv.git cd root2csv pip install --user -r requirements.txt python3 tests/run_tests.py # 36 checks, ~5 s — all green = good to go
On shared servers, pip install --user keeps packages in your home directory — no root, no system changes. A venv works too if you prefer one.
OPTION 1 — FASTEST (one command, default policies)
  1. Point convert.py at the .root file. Scan + flatten run in one shot — the original events.root behavior.
  2. Outputs land in ./<name>_scan/ — see OUTPUTS.
python3 convert.py /data2/kjohns/run3_sample_fastframe_files/<file>.root

Useful extras: --tree reco · --name mySample · --tag is_signal=1 · --fill nan.

OPTION 2 — FULL (worked example, curated policies)
  1. Scan → inspect the census → edit policies → build.
  2. The CSV is rebuilt from the parquet — ROOT is never re-read.
# 1 · scan (all common options shown) python3 scan.py /data2/kjohns/run3_sample_fastframe_files/<file>.root \ --tree reco --name ff01 --out ./ff01_scan \ --tag is_signal=0 --note "fastframe sample, first pass" # 2 · inspect the census (six-bin summary + per-branch stats) less ./ff01_scan/ff01_manifest.txt # 3 · edit per-branch policies in the JSON census, e.g.: # "jet_pt": { ... "policy": "first:6" } keep 6 columns # "trk_d0": { ... "policy": "drop" } exclude from CSV nano ./ff01_scan/ff01_manifest.json # 4 · build the CSV from the parquet, NaN pads python3 convert.py --from-scan ./ff01_scan --fill nan # 5 · results (CSV + build report join the scan artifacts) ls -lh ./ff01_scan/
